# 소개

TON 블록체인에서 스마트 계약 생성, 개발 및 배포는 [FunC 프로그래밍 언어](/develop/smart-contracts/#func-language)와 [TON 가상 머신 (TVM)](/develop/smart-contracts/#ton-virtual-machine)을 활용합니다.

## 빠른 시작: 첫 번째 스마트 계약

_Blueprint_ 프레임워크를 사용하여 첫 번째 스마트 계약을 작성하고 배포하세요.

Blueprint는 스마트 계약을 작성, 테스트 및 배포하는 개발 환경입니다.
새 데모 프로젝트를 만들려면 다음 명령을 사용하세요:

```bash
npm create ton@latest
```

## 스마트 계약 예제

TON 커뮤니티에서 제공하는 준비된 스마트 계약 예제 및 도구를 살펴보세요.

:::info 작은 팁
*FunC*를 사용하여 작성된 스마트 계약에 중점을 두십시오. 종종 낮은 수준의 Fift (_.fif) 언어 대신 FunC (_.fc) 언어를 사용하여 작성된 스마트 계약에 중점을 두는 것이 좋습니다.
:::

TON에서 제공하는 스마트 계약의 표준 예제로는 지갑, 선거인 (TON에서 검증을 관리하는 역할), 다중 서명 지갑 등을 참고할 수 있습니다.

## 커뮤니티 도구

- [disintar/toncli](/develop/smart-contracts/sdk/toncli) — toncli는 FunC 계약을 빌드, 배포 및 테스트하는 데 사용되는 명령줄 인터페이스입니다.
- [MyLocalTON](/participate/run-nodes/local-ton) — MyLocalTON은 로컬 환경에서 개인용 TON 블록체인을 실행하는 데 사용됩니다.
- [tonwhales.com/tools/boc](https://tonwhales.com/tools/boc) — BOC parser.
- [tonwhales.com/tools/introspection-id](https://tonwhales.com/tools/introspection-id) — crc32 생성기.
- [@orbs-network/ton-access](https://www.orbs.com/ton-access/) — 분산형 API 게이트웨이.
